# Project Memory: Opus Form

## System Architecture & State
- **Core**: Local dashboard system. GitHub source control (`dev` branch — lowercase; `/commit` skill default corrected to match). Cloudflare Pages deployment (`nodejs_compat`).
- **Database**: Standalone Supabase (`fgpthpxmiroyebrzjdzo`). Live schemas (profiles, staff, shifts, jobs, quotes, job_attachments, job_diary, job_document_requests). Consolidation of deployments into `shifts` table as the single source of truth; dropped `jobs.assigned_workers` and `shifts.is_removed` columns.
- **Security**: Strict RLS policies. Secure HTTP headers (CSP, MIME protections). Admin-only `public.audit_logs` tracks all system actions. Private storage bucket `job-attachments` with authenticated/anonymous upload RLS.
- **Email**: Resend HTTP API via Supabase Edge Function to bypass anti-bot throttling. `RESEND_API_KEY` lives solely as a Supabase Edge Function secret — the `smtp_config` table row and the `vault.secrets` entry that used to shadow it were both deleted (edge function checks `decrypted_smtp_config` first, which UNIONs `smtp_config` and `vault.decrypted_secrets`, so either one silently overrides the env secret if repopulated). `log_anonymous_audit()` inserts `tenant_id` (derived from the caller's `profiles` row) — omitted after multi-tenancy landed, which threw a not-null violation on every non-whitelisted audit action.

# Project Memory: Opus Form

## System Architecture & State
- **Core**: Local dashboard system. GitHub source control (`dev` branch — lowercase; `/commit` skill default corrected to match). Cloudflare Pages deployment (`nodejs_compat`).
- **Database**: Standalone Supabase (`fgpthpxmiroyebrzjdzo`). Live schemas (profiles, staff, shifts, jobs, quotes, job_attachments, job_diary, job_document_requests). Consolidation of deployments into `shifts` table as the single source of truth; dropped `jobs.assigned_workers` and `shifts.is_removed` columns.
- **Security**: Strict RLS policies. SMTP credentials encrypted in `vault.secrets`. Secure HTTP headers (CSP, MIME protections). Admin-only `public.audit_logs` tracks all system actions. Private storage bucket `job-attachments` with authenticated/anonymous upload RLS.
- **Email**: Resend HTTP API via Supabase Edge Function to bypass anti-bot throttling.

## Core Features
- **Job Ledger & Project Management**: Interactive Job Details dashboard. Live weather via Open-Meteo (keyless, 14-day past + 16-day forecast) geocoded through Nominatim/OSM, cached per postcode and shared across the app (`useJobForecast` hook in `utils/weather.ts`) — no API key stored client-side. Nearby tool-hire/building-merchant supplier search bounded to a ~15mi viewbox around the geocoded site (was an unbounded text match returning irrelevant results). All distances shown in miles. Daily Site Diary notes and digital H&S daily checklists (PPE, briefing, delivery access, weather checks); pour logs disabled unless today's H&S checklist is completed. Before/after photo gallery uploaded to Supabase storage. Drag-and-drop project document attachments. Shareable upload link generation utilizing token-based `public.job_document_requests`.
- **Public Upload Portals**: Passwordless credentials portal at `/submit-credentials` redesigned as a multi-step wizard (deferred upload on submit, local object URL previews, initials filename auto-renaming, and RLS bypass via security definer RPC `get_document_request_details` for staff names). Passwordless job upload portal at `/job-upload/:token` for external document requests.
- **Quote Builder**: Client-side PDF generation (`html2pdf.js` clamped to 794x1122px). Stepper animations, strict UI validation overlays, custom toast modals. Quotes saved to `public.quotes`.
- **Staff Management**: 13 defined roles. Roster list view only with A-Z sorting. Map proximity scheduling (postcode geocoding/Haversine distance). Removed redundant dossier action buttons (Copy Link and Refresh Data). Removed mobile top sub-header, integrated registration button next to search bar (responsive compact icon on mobile, full text on desktop/tablet). Simplified staff audit logs to dot-styled timeline with interactive inline expansion of diffs, revert buttons, and document request links.
- **Compliance & Security**: Passwordless document submission via 48-hour expiring signed URLs to private Supabase bucket. Edge functions enforce JWT & role checks. Data restoration utility built into audit logs with on-screen diff-highlighting confirmation modal (amber strikethrough for changed fields). Detailed changed-fields-only diff tracking for record updates. Logging for dossier inspections. Auto-expiry of all other pending document request links on new request create or resend. Effective resend timestamp derived from `expires_at - 48h` for correct card ordering. Dashboard compliance warnings panel redesigned into compact table-style rows featuring colored left-border severity indicators and integrated inline validation/reminder confirmation modals. Added a dedicated `send-admin-alert` Supabase edge function to handle administrative notifications upon worker email dispatch failures. Logged `COMPLIANCE_REMINDER_SENT` events to track dispatch outcomes. Admin policies dashboard (`/portal/policies`) restricted strictly to `admin@opusform.co.uk`, reading static public URLs from `policies` bucket to bypass RLS enumeration vulnerabilities.
- **UI**: Redesigned Job Details page to match mockup layout (side-by-side Project Status and Pour Progress cards, rain warning banner, and a reversed Pour History log list, removing map and workforce panels). Slide-out drawers for staff dossiers. Unified design systems across staff registry and calendar views (upgraded inputs, date pickers, filter tabs, pagination controls, and control buttons to `rounded-xl` borders, `font-bold`/`font-semibold` weights, and slate-blue `#5C7285` accent colors, replacing chunky `font-black` typography). Event Type mappings on System Audit Trail. Memoized audit diff grid rendering to minimize jank. Touch targets ≥44px on mobile navigation and selectors. Removed mobile/tablet bottom navigation bar and adjusted layout paddings. Removed `/commands` dropdown, matching logic, and updated input placeholder in the Dashboard search. Implemented complete dark portal UI redesign matching the Claude Design handoff (PortalLayout navigation and command top bar, PortalAuth centered login card, PipelineRegistry searchable quote pipeline grid, QuoteInvoiceBuilder split items/summary configuration, and AuditLog severity event timeline). Portal header (desktop sidebar, mobile header, mobile drawer) uses the `opus-form-primary.svg` wordmark logo only — no icon glyph, no "ERP PORTAL" caption. Password recovery labeled as "Password Recovery" with custom redirect listener and session handler for HashRouter. ## Agent Architecture
- **Workspace Customizations**: Workspace instructions, persona switching, mandatory implementation plan + user approval requirements, and manual-verification-only rules (no automated localhost/browser preview) configured in `.agents/AGENTS.md`. Concrete flooring contractor domain identity enforced across internal documentation. Removed all `.claude/` subagents and configuration directories.
- **Skill coverage**: Custom agent skills wired in `.agents/skills/`. Created `dev-server` skill to automate server startup and UI preview guidance.
- **VS Code Tasks**: Integrated `.vscode/tasks.json` configuration with `Open Live Preview` to automatically start the server and load the site inside VS Code Simple Browser on folder open.
- **MCP Servers**: GitHub MCP server configured to run locally via global `npx` command.
- **Global behavioral rules**: Token conservation, terse response style, Composio-availability reminder, spawn-task usage, and UI/UX-skill-check requirement live in `~/.claude/CLAUDE.md` (applies to all projects and subagents), not per-project memory — moved out of OpusForm-specific memory to avoid duplication.

## Roadmap
- **MFA (Supabase TOTP)**: Enable for admin and dispatcher roles. Operatives excluded (field-worker friction). Deferred from v1 — target when pursuing Cyber Essentials Plus.
- **ISO 27001**: Natural progression from Cyber Essentials basic. Pursue when revenue justifies audit cost (£5k–15k).
- **Self-service DSR portal**: Build data subject access/erasure request workflow in portal. Currently manual via admin@opusform.co.uk.
